# نظام حجوزات الشاليهات

تطبيق حجز عربي يعمل من ملف جذر واحد: `/index.html`، بخلفية Supabase (Edge Functions + قاعدة بيانات).

## السطح الرسمي (المنشور على GitHub Pages)

- الصفحة الوحيدة للتطبيق: `/index.html`
- صفحة الخطأ: `/404.html`

مخطط قاعدة البيانات **ليس** جزءاً من نشر الصفحات؛ مصدره الرسمي هو سلسلة الهجرات في
`supabase/migrations/` (تُطبَّق على Supabase عبر ورك‑فلو النشر، لا تُنشَر إلى Pages).
الملف `database/shared_workspace_sync.sql` نسخة مرجعية للأساس (baseline) تبقى مطابقة
للهجرة الأولى فقط، ولا يُنشَر.

## النشر

- **الواجهة (Pages):** ورك‑فلو `Deploy clean root app` يبني `dist` وينشر **فقط**
  `dist/index.html` و`dist/404.html` (يؤكّد أن لا شيء غيرهما) — على كل دفع إلى `main`.
- **الخلفية (Supabase):** ورك‑فلو `Deploy Supabase` (الهجرات + Edge Functions + القياس
  الحيّ) **لا يعمل تلقائياً على كل دمج**. ينشر فقط عند تشغيل يدوي (`workflow_dispatch`)
  أو عند دفعٍ إلى `main` رسالته تحوي العلامة `[deploy]` — حتى لا تُطبَّق هجرة على قاعدة
  البيانات الحيّة دون قصد. راجع `supabase/migrations/README.md`.

> **ملاحظة بيئة (٢٠٢٦‑٠٧):** التطبيق الرسمي يتصل حالياً بمشروع Supabase واحد (نفسه
> المُصنّف `APP_ENV=staging`) وفيه بيانات حقيقية. فصل إنتاج/تجربة حقيقي (مشروعان) بند
> لاحق؛ حتى ذلك، يُعامَل كل نشر خلفية كنشر إنتاج.

## قاعدة التشغيل

- لا تسجيل دخول بالبريد، ولا روابط تحقق بالبريد، ولا مكتبات خارجية.
- الاتصال بالسحابة يتم فقط عبر `fetch` إلى Supabase (RPC + Edge Functions).
- لا رفع تلقائي عند فتح الصفحة أو تغيير التبويبات؛ الرفع فقط من زر «رفع التعديلات».

## الفرع

الفرع الرسمي للنشر: `main`. النسخة الاحتياطية التاريخية: `backup-before-full-clean-rebuild`.
